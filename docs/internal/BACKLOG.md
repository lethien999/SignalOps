## 📋 Process

1. **Ideas** → Added here (Anyone can suggest)
2. **Review** → User decides: **P0 (Critical)**, **P1 (High)**, **P2 (Nice)**, or **Reject**
3. **Approved** → Move to [M10-NewFeatures.md](milestones/M10-NewFeatures.md)
4. **Development** → Tracked in milestone file
5. **Done** → Archive to M1-M9-Completed.md

## Key Backlog Rules

1. **One idea per item** → Easy to track
2. **Brief description** → Use comments for details
3. **Prioritize together** → Not automatic
4. **Deprecate clearly** → Mark ~~strikethrough~~ when rejecting
